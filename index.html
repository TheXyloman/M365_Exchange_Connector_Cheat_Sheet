Microsoft 365 SMTP Relay Runbook

Exchange M365 Connector Scan-to-Email Cheat Sheet

A practical setup and troubleshooting guide for helpdesk engineers configuring printers, scanners, and MFPs to send mail through Exchange Online using an IP-authenticated inbound connector.

Before You Start

  • Customer has a Microsoft 365 tenant with Exchange Online and a verified accepted domain.
  • The device or site has a static public IP address visible to Microsoft 365.
  • Outbound SMTP on TCP port 25 is allowed by firewall, ISP, and any security appliance.
  • You have access to Exchange admin center and the customer public DNS zone.
  • You know the printer web admin password or can reach someone who does.

Quick Reference

        
Known-good SMTP relay values

        

          Use these values when the connector is authenticated by the sending site's public IP address.
          This is for Microsoft 365 SMTP relay, not authenticated SMTP submission.
        

        

          

            Device SMTP settings
            

              
SMTP server
Customer MX endpoint, for example contoso-com.mail.protection.outlook.com

              
Port
25

              
TLS / STARTTLS
Enable where supported. Prefer STARTTLS or TLS 1.2+ capable modes.

              
SMTP authentication
Off / none for IP-authenticated relay.

              
Sender address
Any address in a verified accepted domain, for example scanner@contoso.com.

            

          

          

            Do not mix up
            

              
SMTP relay
Uses MX endpoint, port 25, connector, static public IP.

              
SMTP AUTH
Uses smtp.office365.com, port 587, mailbox credentials, and modern/auth settings.

              
Direct Send
Uses MX endpoint and port 25, but only sends to internal Microsoft 365 recipients.

              
Port 465
Avoid for Microsoft 365 SMTP relay and SMTP AUTH setups.

            

          

        

      


      

        
Step 1

        
Gather the required details

        

          The connector trusts the site by source IP, so the most important fact is the actual public IP that Microsoft 365 sees when the printer sends mail.
        


        
    
          
  1. 
            Confirm the sending location.
            
    Is the printer sending directly from the office, through a print server, through a firewall SMTP proxy, or through another relay?
    
          
    2. 
          
  2. 
            Find the public IP from the same network path.
            
    From a workstation on the same site/VLAN or from the firewall, browse to an IP lookup page or check the firewall WAN interface/NAT rule.
    
          
    3. 
          
  3. 
            Confirm the public IP is static and exclusive.
            
    Dynamic IPs are not suitable. Shared third-party or cloud-hosted sending IPs should not be used for this connector type.
    
          
    4. 
          
  4. 
            Find the MX endpoint for the sending domain.
            
    In Microsoft 365 admin center, go to Settings > Domains, open the verified domain, and copy the Microsoft Exchange MX value.
    
          
    5. 
          
  5. 
            Check the printer basics.
            
    Record printer IP, model, admin URL, DNS server, default gateway, time zone, current date/time, and firmware if available.
    
          
    6. 
        


        

          

            Windows checks
            

              
PowerShell DNS and connectivity checks

              
nslookup -type=mx contoso.com
Resolve-DnsName contoso.com -Type MX
Test-NetConnection contoso-com.mail.protection.outlook.com -Port 25

            

          

          

            macOS checks
            

              
Terminal DNS and connectivity checks

              
dig mx contoso.com
nc -vz contoso-com.mail.protection.outlook.com 25
openssl s_client -starttls smtp -connect contoso-com.mail.protection.outlook.com:25

            

          

        


        

          
Decision point: If the customer only needs to send to internal recipients and cannot create a connector, Direct Send may work, but it will not relay to external addresses. For scan-to-email to external recipients, continue with SMTP relay.

        

      


      

        
Step 2

        
Create the Exchange Online connector

        

          Create an inbound connector in Exchange admin center so Microsoft 365 accepts relay traffic from the customer's static public IP.
        


        
    
          
  1. 
            Open Exchange admin center.
            
    Go to https://admin.cloud.microsoft/exchange#/connectors or open Exchange admin center, then Mail flow > Connectors.
    
          
    2. 
          
  2. 
            Add a connector.
            
    Set Connection from to Your organization's email server. Connection to Office 365 is selected automatically.
    
          
    3. 
          
  3. 
            Name it clearly.
            
    Use a name such as SMTP Relay - Scan to Email - Site Name. Leave Turn it on selected. Do not retain internal Exchange headers unless there is a specific reason.
    
          
    4. 
          
  4. 
            Choose IP authentication.
            
    Select the option to verify that the sending server IP address matches one of the IP addresses belonging exclusively to the organization.
    
          
    5. 
          
  5. 
            Enter the public IP.
            
    Add the static public IP recorded in Step 1. Review the connector and create it.
    
          
    6. 
          
  6. 
            Record the connector details.
            
    Update the worksheet with connector name, public IP, date created, and the engineer who made the change.
    
          
    7. 
        


        

          
Security note: Do not add broad IP ranges unless they are genuinely dedicated to the customer. The connector uses this IP information as an authentication signal.

        

      


      

        
Step 3

        
Update SPF for the sending domain

        

          SPF tells recipients that the customer's public IP is allowed to send mail for the domain. Update the existing SPF TXT record; do not create a second SPF record.
        


        

          

            Example update
            
Before:

            

              
Existing SPF

              
v=spf1 include:spf.protection.outlook.com -all

            

            
After adding the site IP:

            

              
Updated SPF

              
v=spf1 ip4:203.0.113.10 include:spf.protection.outlook.com -all

            

          

          

            SPF checks
            
    
              
  • There must be only one SPF TXT record beginning with v=spf1.
    • 
              
  • Use ip4: for IPv4 addresses and ip6: for IPv6 addresses.
    • 
              
  • Keep DNS lookup limits in mind if the SPF record already has many include: values.
    • 
              
  • Wait for DNS propagation before assuming SPF is broken.
    • 
              
  • Document the original SPF value before changing it.
    • 
            

          

        


        

          
Verify SPF TXT records

          
nslookup -type=txt contoso.com
Resolve-DnsName contoso.com -Type TXT
dig txt contoso.com

        

      


      

        
Step 4

        
Configure the printer or MFP

        

          Vendor screens use different labels, but the settings are usually the same. Configure the device to send directly to the domain MX endpoint over port 25.
        


        

          

            
              

                Setting
                Use this value
                Engineer note
              

            
            
              

                SMTP server / smart host
                contoso-com.mail.protection.outlook.com
                Use the customer's actual MX endpoint, not smtp.office365.com.
              

              

                SMTP port
                25
                Outbound port 25 must be open from the printer or relay path.
              

              

                Connection security
                STARTTLS / TLS enabled where supported
                If the device has old TLS support, update firmware or test without TLS only as a controlled fallback.
              

              

                SMTP authentication
                Disabled / none
                The connector authenticates by public IP, not username and password.
              

              

                From / sender address
                scanner@contoso.com
                Use an address in the verified Microsoft 365 accepted domain.
              

              

                DNS and gateway
                Valid DNS resolver and default gateway
                The device must resolve the MX hostname and route to the internet.
              

              

                Date, time, and timezone
                Correct local time and NTP where possible
                Wrong time can break TLS validation and make logs difficult to read.
              

            
          

        


        
Common printer field mappings

        

          

            HP
            
Look under Scan/Digital Send, Email Setup, or Outgoing Email Profiles. Set SMTP Server, SMTP Port, STARTTLS/TLS, Default From, and SMTP Authentication off.

          

          

            Lexmark
            
Look under Settings > Email/FTP Settings > SMTP Setup. Set Primary SMTP Gateway, SMTP Port, Use SSL/TLS or STARTTLS, Reply Address, and Device Userid/Password blank if auth is disabled.

          

          

            Canon
            
Look under Network Settings, TX Settings, or E-Mail/I-Fax Settings. Set SMTP Server, E-Mail Address, SMTP Receive/Send settings, STARTTLS if available, and SMTP Authentication off.

          

          

            Brother
            
Look under Network > Protocol > SMTP or Scan to E-mail Server. Set SMTP Server Address, Port, SSL/TLS or STARTTLS, sender address, and Authentication Method to none/no auth.

          

        


        

          
Practical tip: After saving SMTP settings, restart the printer if tests keep using old values. Some devices cache DNS or SMTP settings until a reboot.

        

      


      

        
Step 5

        
Test the configuration

        

          Test from the network first, then from the printer. A successful connection test does not prove delivery, so always send a real message and verify it arrives.
        


        

          

            Windows PuTTY raw test
            
    
              
  1. Open PuTTY.
    2. 
              
  2. Set Connection type to Raw.
    3. 
              
  3. Host Name: customer MX endpoint.
    4. 
              
  4. Port: 25.
    5. 
              
  5. Open the connection and confirm the SMTP banner appears.
    6. 
            

            

              
Manual SMTP conversation

              
EHLO test.contoso.com
MAIL FROM:<scanner@contoso.com>
RCPT TO:<testuser@contoso.com>
DATA
Subject: Scan to email connector test

This is a manual SMTP relay test.
.
QUIT

            

          

          

            macOS Terminal test
            

              
Connectivity and STARTTLS

              
nc -vz contoso-com.mail.protection.outlook.com 25
openssl s_client -starttls smtp -connect contoso-com.mail.protection.outlook.com:25

            

            
Use the printer test function after confirming port 25 is reachable. Confirm the message arrives in Inbox, Junk, or quarantine.

          

        


        

          
Pass criteria: The printer sends to an internal Microsoft 365 recipient and an approved external recipient, the message is traceable in Exchange admin center, and headers show the expected sending public IP.

        

      


      

        
Step 6

        
Troubleshooting matrix

        

          Work from network path to Microsoft 365 configuration to printer settings. Record each finding so the next engineer does not repeat the same test.
        


        

          

            
              

                Symptom
                Likely cause
                What to check
                Fix
              

            
            
              

                Connector rejects or mail is not accepted
                Microsoft 365 sees a different source IP
                Message trace, firewall NAT, WAN IP, SMTP proxy path
                Add the correct static public IP or fix NAT/routing.
              

              

                Cannot connect to MX on port 25
                Firewall, ISP, or security service blocks SMTP
                Test-NetConnection, nc, firewall logs, ISP policy
                Allow outbound TCP 25 from the printer/relay to Exchange Online Protection endpoints.
              

              

                Printer test says DNS failed
                Bad DNS server, gateway, or hostname typo
                Printer network page, DNS lookup from same subnet
                Correct DNS/gateway and use the exact MX endpoint.
              

              

                External recipients fail but internal works
                Device is using Direct Send, not SMTP relay
                Connector exists, host is MX endpoint, source IP is trusted
                Create/fix the inbound connector and retest external delivery.
              

              

                Mail lands in Junk or quarantine
                SPF missing, duplicated, or not propagated
                TXT records, message headers, Defender quarantine
                Update the single SPF record and allow DNS propagation.
              

              

                TLS or certificate errors
                Old firmware, wrong time/date, unsupported TLS
                Device clock, firmware, TLS setting, OpenSSL output
                Correct time, enable NTP, update firmware, use STARTTLS where possible.
              

              

                Sender address rejected
                From address domain is not accepted/verified
                Microsoft 365 accepted domains and printer sender field
                Use a sender address in a verified accepted domain.
              

              

                Setup worked previously then stopped
                ISP/router/firewall public IP changed
                Current WAN IP, connector IP, SPF IP
                Restore static IP or update connector and SPF with change control.
              

              

                SMTP banner appears but DATA fails
                Relay restriction, sender/recipient issue, malformed test
                SMTP response code, accepted domain, recipient address
                Use valid addresses and compare against message trace.
              

            
          

        

      


      

        
Step 7

        
Escalation notes

        

          Escalate with evidence. The goal is to show which layer failed: device, DNS, firewall, Microsoft 365 connector, SPF, or recipient filtering.
        


        

          

            Firewall team
            
Provide printer IP, destination MX hostname, destination port 25, timestamp, and firewall logs showing allow/deny/NAT source IP.

          

          

            M365 admin
            
Provide connector name, public IP, sender, recipient, time sent, SMTP error, and message trace result.

          

          

            Printer support
            
Provide model, firmware, SMTP settings screenshot, test result, DNS/gateway settings, date/time, and whether STARTTLS is supported.

          

        


        

          
Change control: If the public IP changes, update both the Exchange connector and SPF record. Test internal and external delivery again after DNS propagation.
